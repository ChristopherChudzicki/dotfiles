## Decisions and context not in the spec/plan

- **User is on Apple Silicon Mac.** Brew prefix is `/opt/homebrew`. Plan assumes this throughout.
- **A second Mac is incoming** (also Apple Silicon expected). The repo is designed to be cloned and `install.sh`-ed there.
- **Login shell stays `zsh`** through this entire effort. Fish is opt-in via the `fish` command. No `chsh`. No edits to `/etc/shells`.
- **`pyenv` is being dropped** entirely. User uses `uv` exclusively for Python now. The pyenv install on disk is left untouched but no longer initialized in either shell.
- **`nvm` is being replaced by `fnm`.** `~/.nvm/` is left on disk as a fallback during transition. User will re-install needed Node versions via `fnm install <version>`. Removal of `~/.nvm` is the user's call later.
- **`ocw_studio_env` zsh function is dropped.** No longer relevant.
- **`ghc` zsh function is preserved** — translated to fish in `fish/functions/ghc.fish`.
- **`chrome` and `chrome-debug` aliases are preserved** in zsh and become fish functions.
- **Starship config is hand-written** in the plan (Task 5) rather than seeded from a preset, because preset names were uncertain. The hand-written version covers the same ground (two-line, git branch + status, language versions). If a preset fits better, swapping later is fine.
- **Subagent execution is fine for most tasks.** The plan was written for subagent-driven execution — each task is independently dispatchable. Tasks 7–10 modify the user's actual shell environment (with backups via `install.sh`); tasks 9 and 11 need user input.

## Permission notes

A fresh `claude` session in this repo will likely still need approval for:
- `brew install` (Task 3)
- Symlink creation in `~/.config/`, `~/.zshrc`, `~/.zprofile` (Task 7 via `install.sh`)
- `fnm install` downloads (Task 9)
- `gh repo create` and the first `git push` (Task 11)

These are expected and the user has pre-approved them in concept by approving the plan. Approval is a per-command prompt thing in Claude Code.

## Manual checkpoints (will need user input)

- **Task 9** — needs to know which Node versions to install. The plan says: list `~/.nvm/versions/node/`, install each in `fnm`, ask user which should be the default.
- **Task 11** — needs user confirmation before creating the GitHub remote (public vs private; repo name confirmation).

## Rollback (if needed)

If anything goes wrong after `install.sh` runs:
- Backups of original `.zshrc` and `.zprofile` are at `~/dotfiles-backup/2026-04-25/`
- Restore: `rm ~/.zshrc ~/.zprofile && cp ~/dotfiles-backup/2026-04-25/.zshrc ~/dotfiles-backup/2026-04-25/.zprofile ~/`
- The brew installs are independent — each can be removed with `brew uninstall <pkg>` if needed
- `~/.nvm/` and pyenv install are untouched
